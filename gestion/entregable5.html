5.1 Arquitectura de la Solución

El proyecto emplea una arquitectura estructurada basada en microservicios y despliegue local (On-Premise) para garantizar un procesamiento biométrico de baja latencia.

📱
Capa de Presentación
Aplicación móvil en Flutter 3 con interfaz asíncrona para el docente (monitoreo de reconocimiento y reportes con "fricción cero").
⚙️
Capa de Negocio (Orquestador)
Core API en Laravel 11. Maneja autenticación JWT, reglas académicas, CRUD de estudiantes y registro de asistencias.
🧠
Capa de Inteligencia Artificial
Microservicio en Python (FastAPI) dedicado a la inferencia en tiempo real con YOLOv8 (detección) y ArcFace R100 (vectorización).
💾
Capa de Centro de Datos
Entorno containerizado local mediante Ubuntu Server + Docker, con almacenamiento relacional en MySQL 8.
🌐
Capa de Red
Topología LAN (Air-gapped by design) configurada con Switch, Firewall (reglas ACL) bajo el estándar TIA/EIA 568.

5.2 Ecosistema e Integración de Sistemas

La solución está diseñada para interoperar de manera transparente con el ecosistema de Tecnologías de Información (ERP/Sistema Académico) actual de la UPeU.

🗺️
Mapa de Integración
El flujo de video crudo capturado por las cámaras IP nunca abandona la red LAN local del colegio, garantizando un aislamiento total de los datos biométricos.
🔗
Interoperabilidad
Módulo autónomo que expone servicios mediante API REST documentada con Swagger. El sistema central de la universidad consume reportes en JSON para libretas o métricas de retención.
☁️
Arquitectura Híbrida Segura
Nube (AWS S3 / Google Drive) únicamente para sincronizar reportes administrativos anonimizados y respaldos cifrados, logrando cero exposición biométrica al exterior.

5.3 Seguridad y Gobernanza de Datos

El sistema cumple estrictamente con el marco normativo ISO 27001 y los lineamientos de la Ética ACM para la protección absoluta de datos de estudiantes menores de edad.

🔐
Protección de Datos (Digital Vault)
Las fotografías originales se procesan en memoria RAM y se destruyen inmediatamente. En BD se almacenan exclusivamente vectores matemáticos (embeddings de 512 dimensiones) cifrados localmente, haciendo imposible la ingeniería inversa para reconstruir un rostro.
🛡️
Gestión de Accesos
Acceso al sistema y a los reportes estrictamente restringido mediante Listas de Control de Acceso (ACLs) y validación de tokens JWT en cada petición.

5.4 Sustento Técnico de Escalabilidad y Sostenibilidad

La viabilidad tecnológica (GTI) del ecosistema se sustenta en una fuerte optimización de recursos CapEx y OpEx, garantizando su sostenibilidad a largo plazo en la institución.

🔄
Evolución Tecnológica
Enrollment Dinámico: El modelo IA está preentrenado con el dataset MS1M-V2 (5.8M imágenes). Permite matricular nuevos estudiantes cada año sin volver a entrenar la red neuronal.
📈
Capacidad de Crecimiento
Arquitectura Docker permite escalabilidad horizontal. Si la institución amplía aulas, basta con desplegar contenedores adicionales del microservicio FastAPI sin modificar el código base.
💰
Costos Futuros Controlados
$0 en licencias (stack 100% Open Source). ArcFace R100 se ejecuta en CPU estándar (i5/Ryzen, 8GB RAM), eliminando la necesidad de GPU costosa para futuras expansiones.
