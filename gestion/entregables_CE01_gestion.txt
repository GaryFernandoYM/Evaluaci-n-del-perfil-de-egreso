2. DESCRIPCIÓN DEL PROYECTO
-----------------------------
QhawAI es un sistema de control de asistencia basado en reconocimiento facial,
desarrollado para la Institución Educativa [FALTA COMPLEMENTAR]. El sistema
automatiza el registro de ingreso de estudiantes y docentes mediante cámaras
y algoritmos de inteligencia artificial.

Componentes del sistema:
  - Backend web:     Laravel 13.5 (PHP 8.3)
  - IA / Reconoc.:   FastAPI + DeepFace (Python)
  - Base de datos:   MySQL
  - App móvil:       Android (Kotlin)
  - Autenticación:   Firebase Auth

2. CATÁLOGO DE PROCESOS
-------------------------
El sistema QhawAI gestiona los siguientes procesos principales:

  PRO-01: Registro de asistencia por reconocimiento facial
    Actores:   Sistema (cámara + IA), Docente, Base de datos
    Disparador: Inicio de clase / hora de entrada
    Resultado: Registro automático en BD, notificación al docente

  PRO-02: Generación de reportes de asistencia
    Actores:   Administrador, Director, Sistema web
    Disparador: Solicitud de reporte (diario/semanal/mensual)
    Resultado: PDF/Excel con resumen de asistencias y faltas

  PRO-03: Gestión de personas (alumnos y docentes)
    Actores:   Administrador
    Disparador: Matrícula o cambio de personal
    Resultado: Persona registrada con foto para entrenamiento IA

  PRO-04: Entrenamiento del modelo facial
    Actores:   Sistema IA (FastAPI), Administrador
    Disparador: Nuevas fotos cargadas al sistema
    Resultado: Modelo actualizado, mayor precisión de reconocimiento

  PRO-05: Notificaciones a padres / apoderados
    Actores:   Sistema, Firebase Cloud Messaging
    Disparador: Registro de falta o tardanza
    Resultado: Notificación push al apoderado

3. DIAGRAMA DE FLUJO DE PROCESO PRINCIPAL
-------------------------------------------
[ INSERTAR IMAGEN: Diagrama BPMN del proceso PRO-01 (registro facial) ]

Descripción del flujo PRO-01:
  1. Alumno/docente se presenta ante la cámara
  2. Sistema captura frame de video
  3. FastAPI detecta y codifica rostro
  4. Sistema compara con embeddings registrados
  5a. Si coincide (similitud > 70%): registra asistencia en MySQL
  5b. Si no coincide: registra como "desconocido", alerta al docente
  6. Laravel muestra el registro en tiempo real en el dashboard
